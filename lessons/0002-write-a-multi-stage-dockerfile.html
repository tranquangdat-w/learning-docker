Write Your First Multi-Stage Dockerfile

Lesson 0002 · you are given a Node app — you write the Dockerfile

The scenario

You have a simple Express server in exercises/node-express/:

const express = require('express');
const app = express();
const port = process.env.PORT || 3000;

app.get('/', (req, res) => {
  res.json({ message: 'Hello from Node!' });
});

app.listen(port, () => {
  console.log(`Server running on port ${port}`);
});

It has one dependency: express. Right now the node_modules don't exist — they get created when you npm install.

Your task

Write a Dockerfile in exercises/node-express/ with two stages:

Stage 1 — deps

Base: node:20-alpine. Copy package.json, install only production dependencies (npm ci --only=production), then clean the npm cache.

Stage 2 — release

Base: node:20-alpine. Copy node_modules from deps, then copy server.js. Expose port 3000 and run with node server.js.

Hints

  • Every stage starts with FROM. You need two.
  • Name the first stage with AS deps so the second can reference it.
  • npm cache clean --force keeps cache out of the image layer.
  • The final stage copies only the two things it needs: node_modules and the source file.

Solution

Reveal after you’ve tried it 
FROM node:20-alpine AS deps
WORKDIR /app
COPY package.json ./
RUN npm ci --only=production && npm cache clean --force

FROM node:20-alpine AS release
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY server.js ./
EXPOSE 3000
CMD ["node", "server.js"]
Why two stages even though both use the same base image? The deps stage builds node_modules and then discards the npm cache and any temporary install files. The release stage gets a clean node_modules without the install debris. On a real project with devDependencies (TypeScript, webpack), the second stage wouldn't even need those installed — only what COPY --from brings in.

Primary source

Docker’s own best-practices guide for Node: docs.docker.com/guide/nodejs. It shows a similar two-stage pattern for a real Node service.
